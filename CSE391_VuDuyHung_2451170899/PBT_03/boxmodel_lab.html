📦 Box Model Lab - Content-box vs Border-box

Chứng minh sự khác biệt giữa hai cách tính box model trong CSS

Phần 1: So sánh Content-box vs Border-box

ℹ️ Mở DevTools (F12) → Chọn từng hộp → Tab "Computed" → Xem "Box Model diagram"

Box 1: content-box

width: 300px

padding: 20px

border: 5px

Box 2: border-box

width: 300px

padding: 20px

border: 5px

📐 Tính toán:

Box 1 (content-box):
  • Chiều rộng hiển thị = 300 + (20×2) + (5×2) = 350px
  • Không gian chiếm = 350 + (10×2) margin = 370px
Box 2 (border-box):
  • Chiều rộng hiển thị = 300px (bao gồm padding + border)
  • Content thực = 300 - (20×2) - (5×2) = 250px
  • Không gian chiếm = 300 + (10×2) margin = 320px
💡 Kết luận: Vì `box-sizing: border-box`, chiều rộng thực tế không tăng thêm khi thêm padding/border. Đây là lý do tại sao nên dùng `* { box-sizing: border-box; }` ở dầu mọi dự án CSS!

Phần 2: Layout 3 Cột (Content-box Problem)

ℹ️ Container rộng 1000px. 3 cột phải nằm cạnh nhau mà không wrap.

❌ Cách SAIIII (Không dùng border-box)

Sidebar
250px + 30px padding + 2px border = 282px
Content
500px + 40px padding + 2px border = 542px
Ads
250px + 30px padding + 2px border = 282px

🔴 Tổng = 282 + 542 + 282 = 1106px > 1000px → Content bị wrap xuống!

✅ Cách ĐÚNG (Dùng border-box)

Sidebar
width: 250px
(bao gồm padding + border)
Content
width: 500px
(bao gồm padding + border)
Ads
width: 250px
(bao gồm padding + border)

🟢 Tổng = 250 + 500 + 250 = 1000px ✓ Perfect fit!

💡 Giải pháp: Thêm `* { box-sizing: border-box; }` ở đầu CSS. Khi đó, width đã bao gồm padding + border, không cần tính toán phức tạp.

Phần 3: Margin Collapse

ℹ️ Khoảng cách giữa 2 block elements không CỘNG, mà lấy giá trị LỚN HƠN

Box A: margin-bottom = 30px
Box B: margin-top = 50px

📐 Tính toán Margin Collapse:

  • Box A: margin-bottom = 30px
  • Box B: margin-top = 50px
  • Khoảng cách = max(30, 50) = 50px
  • ❌ KHÔNG PHẢI 30 + 50 = 80px
  • ✅ Margin collapse: Lấy cái lớn nhất!
💡 Quy tắc Margin Collapse:
  • ✓ Áp dụng cho block elements
  • ✓ Chỉ vertical margins (top/bottom), không horizontal
  • ✓ Margin của 2 elements kề nhau sẽ "collapse"
  • ✓ Lấy max value, không cộng lại
  • ✗ KHÔNG áp dụng nếu có padding/border/float giữa chúng
