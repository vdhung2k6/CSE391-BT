🔧 Debug Layout - CSS Float Problem & Solutions

Container rộng 960px, sidebar + content bị wrap xuống

❌ PROBLEM: Layout bị vỡ (content-box)

Container: 960px | Sidebar: 300px + 20px padding + 1px border | Content: 660px + 30px padding + 1px border
Tổng cộng: 342px + 722px = 1064px > 960px → Content wrap xuống!

Sidebar

width: 300px

padding: 20px

border: 1px

= 342px

Content

width: 660px

padding: 30px

border: 1px

= 722px

🔴 Chiều rộng thực tế: 342 + 722 = 1064px (VƯỢT QUÁ 960px!)

📐 Tính toán chi tiết (content-box):

  • Sidebar: 300 + (20×2) + (1×2) = 300 + 40 + 2 = 342px
  • Content: 660 + (30×2) + (1×2) = 660 + 60 + 2 = 722px
  • Tổng: 342 + 722 = 1064px
  • Container: 960px
  • Vượt quá: 1064 - 960 = 104px → Content bị wrap!

✅ SOLUTION 1: Dùng border-box (KHUYẾN NGHỊ)

Thêm `* { box-sizing: border-box; }` → Width đã bao gồm padding + border!

Sidebar

width: 250px

(đã bao gồm padding + border)

Content

width: 500px

(đã bao gồm padding + border)

Ads

width: 250px

(đã bao gồm padding + border)

🟢 Chiều rộng thực tế: 250 + 500 + 250 = 960px ✓ PERFECT!

💻 CSS Code:

/* BƯỚC 1: Reset tất cả elements */
* {
    box-sizing: border-box;
}

.container {
    width: 960px;
    margin: 0 auto;
}

.sidebar {
    width: 250px;      /* Width đã = padding + border + content */
    padding: 20px;
    border: 1px solid #ccc;
    float: left;
}

.content {
    width: 500px;      /* Width đã = padding + border + content */
    padding: 30px;
    border: 1px solid #ccc;
    float: left;
}

.ads {
    width: 250px;      /* Width đã = padding + border + content */
    padding: 20px;
    border: 1px solid #ccc;
    float: left;
}

✅ Ưu điểm:

  • Đơn giản, dễ hiểu
  • Không cần tính toán phức tạp
  • Thích ứng tốt khi thay đổi padding
  • Nên dùng từ đầu mọi dự án CSS

✅ SOLUTION 2: Không dùng border-box, điều chỉnh width

Tính toán lại width để phù hợp: width = (target size) - padding - border

Sidebar

width: 208px

padding: 20px

border: 1px

Content

width: 428px

padding: 30px

border: 1px

Ads

width: 208px

padding: 20px

border: 1px

🟢 Chiều rộng thực tế: 250 + 500 + 250 = 960px ✓ PERFECT!

💻 CSS Code:

/* KHÔNG dùng border-box */
.container {
    width: 960px;
    margin: 0 auto;
}

.sidebar {
    /* Target width = 250px */
    /* 250 - (20×2) - (1×2) = 250 - 40 - 2 = 208px */
    width: 208px;
    padding: 20px;
    border: 1px solid #ccc;
    float: left;
}

.content {
    /* Target width = 500px */
    /* 500 - (30×2) - (1×2) = 500 - 60 - 2 = 428px */
    width: 428px;
    padding: 30px;
    border: 1px solid #ccc;
    float: left;
}

.ads {
    /* Target width = 250px */
    /* 250 - (20×2) - (1×2) = 250 - 40 - 2 = 208px */
    width: 208px;
    padding: 20px;
    border: 1px solid #ccc;
    float: left;
}

❌ Nhược điểm:

  • Tính toán phức tạp, dễ nhầm lẫn
  • Khó maintain khi thay đổi padding/border
  • Nếu quên tính, layout sẽ bị vỡ

⚖️ So sánh 2 cách sửa:

Tiêu chí Cách 1: border-box Cách 2: Điều chỉnh width
Độ phức tạp ✅ Đơn giản ❌ Phức tạp
Tính toán ✅ Không cần ❌ Cần tính toán
Maintain ✅ Dễ ❌ Khó
Thay đổi padding ✅ Không ảnh hưởng layout ❌ Phải tính lại width
Khuyến nghị ✅ DÙNG CÁI NÀY ❌ Tránh nếu có thể

🎯 Kết luận:

Dùng Cách 1 (border-box) từ đầu mọi dự án!

Thêm dòng này vào đầu CSS file của bạn: 

* {
    box-sizing: border-box;
}

Điều này sẽ tiết kiệm rất nhiều thời gian debug layout issues.

💡 Tips & Best Practices:

1. Float Layout (Legacy)

Float dùng để tạo layout cột. Tuy nhiên, hiện tại nên dùng:

  • Flexbox (dễ dùng, tương thích tốt)
  • CSS Grid (mạnh mẽ, cho layout phức tạp)

2. Box Model là nền tảng

Hiểu rõ box model giúp bạn:

  • Debug layout issues nhanh hơn
  • Tính toán sizing chính xác
  • Tránh các bugs liên quan đến spacing

3. Clear-fix for Float

Khi dùng float, phần tử sau float sẽ "wrap" vào bên cạnh. Cách sửa: 

.clear {
    clear: both;
}
