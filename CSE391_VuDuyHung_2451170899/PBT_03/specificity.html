⚔️ Specificity Battle - CSS Cascade War

10 CSS rules khác nhau cùng target 1 element

🎯 Element bị "tấn công" bởi 10 CSS rules:

Hello World

Element: <p id="demo" class="text highlight">Hello World</p>

Mục đích: Chứng minh specificity quyết định rule nào được áp dụng cuối cùng.

📋 10 CSS Rules (từ LOW → HIGH Specificity):

STT CSS Rule Specificity Màu
1 * { color: black; } (0, 0, 0) 🔵 Black
2 p { color: navy; } (0, 0, 1) 🔵 Navy
3 .text { color: blue; } (0, 1, 0) 🔵 Blue
4 .highlight { color: purple; } (0, 1, 0) 🟣 Purple
5 p.text { color: cyan; } (0, 1, 1) 🔵 Cyan
6 .text.highlight { color: lime; } (0, 2, 0) 🟢 Lime
7 p.text.highlight { color: red; } (0, 2, 1) 🔴 Red
8 #demo { color: orange; } (1, 0, 0) 🟠 Orange
9 #demo.text { color: gold; } (1, 1, 0) 🟡 Gold
10 #demo.text.highlight { color: green; } (1, 2, 0) 🟢 Green (WINNER!)

🏆 Kết quả:

Element "Hello World" hiển thị:

Màu GREEN (từ rule #10: #demo.text.highlight)

💡 Giải thích:

  • Rule #10 thắng vì: Có specificity cao nhất (1, 2, 0)
  • Specificity so sánh thứ tự:
    1. ID selectors (a) - cao nhất
    2. Class selectors (b) - thứ hai
    3. Element selectors (c) - thấp nhất
  • So sánh: Rule #10 (1,2,0) > Rule #9 (1,1,0) > Rule #8 (1,0,0) > Rule #7 (0,2,1) > ...
  • Cascade logic: Khi multiple rules áp dụng, rule có specificity cao nhất thắng
  • Thứ tự trong CSS file: KHÔNG quan trọng nếu specificity khác nhau

🔄 Thay đổi thứ tự CSS rules - Kết quả có đổi không?

✅ Kết luận: KHÔNG thay đổi!

Vì specificity của rule #10 (1,2,0) luôn cao hơn các rule khác, dù rule #10 ở dòng 1 hay dòng cuối cùng trong CSS file, nó vẫn luôn được áp dụng (vẫn hiển thị GREEN).

Ngoại lệ: Chỉ khi 2 rules có specificity BẰNG NHAU thì thứ tự CSS mới quan trọng (rule được viết sau sẽ override rule trước).

Ví dụ - 2 rules cùng specificity:

.text.highlight { color: red; }

.text.highlight { color: blue; }

→ Kết quả: BLUE (vì rule sau override rule trước)

🎓 Tips & Best Practices:

1. Tối ưu Specificity:

Đừng dùng ID selector quá nhiều. Thích dùng class selectors để dễ override khi cần.

2. Tránh !important:

!important là cách "phá vỡ" cascade. Nó làm CSS khó maintain. Chỉ dùng khi thật cần thiết.

3. Thứ tự CSS file:

Nếu 2 rules có specificity bằng nhau, rule sau sẽ override rule trước. Vì vậy hãy viết CSS cụ thể/đặc biệt ở cuối.

🔧 Sử dụng DevTools để kiểm chứng:

  1. Mở DevTools (F12) → Inspector (Ctrl+Shift+C)
  2. Click vào element "Hello World"
  3. Tab "Styles" sẽ hiển thị tất cả CSS rules áp dụng
  4. Rules bị "strike-through" là những rules bị override
  5. Rule hiển thị bình thường (không strike-through) là rule được áp dụng cuối cùng
